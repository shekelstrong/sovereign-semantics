Новый проект на `*.vercel.app` ждёт 2-6 недель до первой индексации Google. Тот же контент на собственном домене попадает в индекс за 1-3 дня. Разница не в хостинге. Разница в доменном суффиксе, который Google классифицирует как low-trust shared resource. Цена устранения проблемы: 100-150 ₽ в месяц.

В 2024-2026 годах Google усилил классификатор для shared-доменов. Под демotion попали `*.vercel.app`, `*.netlify.app`, `*.onrender.com`, `*.herokuapp.com`, `*.repl.co`, `*.glitch.me` и частично `*.github.io`. Алгоритм не штрафует эти домены явно. Он снижает приоритет в очередях краулинга и поиска. Страница на vercel.app живёт в полуиндексе: Google её видит, но не показывает.

Новая статья на vercel.app попадает в индекс за 14-45 дней. На собственном домене за 1-5 дней. Discover и News для shared-доменов закрыты. Яндекс мягче, но `.vercel.app` ранжируется ниже `.ru` и `.com` в коммерческих запросах. Отдельная проблема: бренд-каннибализация. Пользователи запоминают «vercel app какой-то», а не название проекта.

Хостинг при этом работает безупречно. Глобальный CDN, нулевая цена на Hobby-плане. Проблема исключительно в доменном имени. Google знает: по адресу `*.vercel.app` завтра может оказаться другой контент. Проекты на shared-доменах имеют высокую churn rate. Создаются и бросаются. Алгоритм реагирует соответственно.

## Что видит Googlebot

Googlebot заходит на `sovereign-semantics.vercel.app/blog/vpn-crypto-2026` и разбирает URL послойно. Subdomain `sovereign-semantics` уникален. Public suffix `vercel.app` помечен как shared, low-trust. Здесь срабатывает классификатор для массовых хостингов.

IP 76.76.21.21 принадлежит Vercel shared range. Reverse DNS `cname.vercel-dns.com` идентифицирует CDN. SSL валидный, контент свежий, backlinks единичные, domain age 0 дней. Вердикт алгоритма: конец очереди. Проект существует в полуиндексе. Проиндексирован, но не показан.

После переезда на `sovereign-semantics.ru` картина меняется. Public suffix `sovereign-semantics.ru` authoritative, high-trust. IP проходит через Cloudflare proxy и выглядит как самостоятельный хостинг. Reverse DNS Cloudflare. SSL валидный. Контент идентичный. Backlinks те же плюс вес с 301-редиректа. Domain age 0 дней, sandbox присутствует, но без shared demotion.

Результат по данным 2024-2026 годов. Первая страница в индексе за 1-5 дней вместо 14-45. Рост в топ-10 за 3-8 недель вместо 6-12 месяцев. Попадание в Discover в течение месяца после набора критической массы страниц.

## Протокол миграции

Preview-домен допустим для хакатонов и MVP за 48 часов с последующим переездом. Для внутренних инструментов команды без публичного доступа. Для proof of concept на столе инвестору с переездом перед запуском. Для pet project без коммерческих целей.

Preview-домен недопустим для стартапа, привлекающего инвестиции: инвестор проверяет домен первым делом. Для блога или медиа, рассчитанного на органический трафик. Для продукта с коммерческой монетизацией. Для проекта в сфере финансов, здоровья или юриспруденции, где аудитория оценивает доверие по адресу.

Выбор домена. Короткий, до 12 символов без зоны. Произносимый, без цифр, дефисов, удвоенных согласных. Без двусмысленностей при чтении вслух. Без конфликтов с товарными знаками: проверка через WIPO Global Brand Database и базы Роспатента. Доступный во всех нужных зонах: `.ru`, `.com`, `.рф` купить сразу все три и настроить редирект на основной. Антипаттерны 2026: `mybusiness-website-2026-online.ru` невозможно запомнить, `mbr2026.online` выглядит как дорвей, punycode `xn--...` невозможно прочитать.

Регистрация. Cloudflare Registrar для `.com/.net/.org/.io`: цена без наценки, бесплатный whois-privacy, интеграция с DNS. reg.ru для `.ru/.рф`: поддержка кириллических зон, удобный интерфейс, но наценка на продление и платный whois-privacy. Для sovereign-проекта: Cloudflare Registrar для `.com`, reg.ru для `.ru` и `.рф`. Все три домена, редирект `www` и apex на основной.

| Регистратор | Плюсы | Минусы | Зоны |
|---|---|---|---|
| Cloudflare Registrar | Цена без наценки, whois-privacy бесплатный | Ограниченный набор зон | `.com/.net/.org/.io` |
| reg.ru | Поддержка `.ru/.рф`, удобный UI | Наценка на renewal, whois-privacy платный | `.ru/.рф` |
| nic.ru | Официальный партнёр RU-Center | Дорого, интерфейс 2000-х | Legacy |
| Namecheap | Хороший UI, whois-privacy бесплатный | Дорого на renewal | Универсальный |
| Porkbun | Дешёвые TLD (.xyz, .dev, .app) | Меньше зон | Специфические TLD |

Настройка DNS. Минимальный набор записей:

```
A     @           76.76.21.21          (Vercel apex)
CNAME www         cname.vercel-dns.com (Vercel www)
MX    @           10 mx.yandex.ru       (почта)
TXT   @           "v=spf1 include:_spf.yandex.net ~all"
TXT   _dmarc      "v=DMARC1; p=quarantine; rua=mailto:admin@..."
```

Cloudflare Proxy включается оранжевым облаком для всех записей: бесплатный SSL, скрытие origin IP, базовая защита от DDoS, кеширование на edge. При включении Cloudflare Proxy нужно отключить встроенный DDoS в Vercel (Settings → Domains), иначе конфликт двух защитных слоёв.

Подключение к Vercel. Settings → Domains → Add Domain, ввести `sovereign-semantics.com` (или `.ru`). Vercel выдаёт DNS-записи для добавления у регистратора. При использовании Cloudflare сначала выставить «DNS only» (серое облако) для подтверждения владения, потом переключить на «Proxied» (оранжевое). SSL Vercel выдаёт через Let's Encrypt автоматически за 5-10 минут. После подключения проверить все поддомены, активные на vercel.app: старые URL не должны отдавать 404, нужен 301-редирект.

Редирект с vercel.app. В Vercel: Settings → Domains → для каждого проекта добавить `old-project.vercel.app` → 301 redirect → `sovereign-semantics.com`. Или в `vercel.json`:

```json
{
  "redirects": [
    {
      "source": "/:path*",
      "destination": "https://sovereign-semantics.com/:path*",
      "statusCode": 301
    }
  ]
}
```

В Cloudflare: Rules → Page Rules → `*sovereign-semantics.vercel.app/*` → 301 Redirect → `https://sovereign-semantics.com/$1`. Только 301, не 302. Постоянный редирект передаёт PageRank, временный не передаёт.

Sitemap, IndexNow и Search Console. В `src/app/sitemap.ts` заменить `SITE_URL` на новый домен. IndexNow submit сразу после деплоя: POST на `https://api.indexnow.org/indexnow` с JSON-списком топ-20 URL. Google Search Console: добавить новый домен как ресурс, отправить sitemap, нажать «Запросить индексирование» для топ-5 URL. То же в Яндекс.Вебмастере. IndexNow работает в Bing, Yandex, Naver, Seznam и Google (с 2025 года), отрабатывает за 1-3 дня.

Ошибки миграции. Переезд без 301-редиректа: потеря всего накопленного PageRank. Переезд с 302-редиректом: Google не передаёт вес и индексирует обе версии. Смена контента во время переезда: алгоритм считает это новым сайтом, sandbox стартует заново. Забытый canonical в Open Graph: og:image со старого домена не отдаётся, шеринг в соцсетях ломается. Не настроенный HTTPS HSTS: браузер ходит на HTTP по привычке. Забытые email-адреса в подписках, GSC, Яндекс.Вебмастере и аналитике.

Чек-лист первого дня после переезда. DNS A и CNAME записи у регистратора. Cloudflare Proxy включён, SSL Full (Strict). Vercel: домен добавлен, SSL активен. 301-редирект со всех старых vercel.app URL. sitemap.xml обновлён на новый домен. robots.txt обновлён. Open Graph og:url и og:image на новом домене. canonical URL на новом домене. GSC: новый ресурс добавлен, sitemap отправлен. Яндекс.Вебмастер: новый ресурс добавлен, sitemap отправлен. IndexNow: топ-20 URL отправлены. Email на новом домене настроен. GA4 и Яндекс.Метрика: allowed_domains обновлены. Все ссылки в соцсетях, профилях и подписях обновлены.

## Что это значит

Собственный домен в 2026: инвестиция с ROI от 1000% для любого публичного проекта. 100-150 ₽ в месяц за полный контроль над брендом, доверие поисковиков, защиту от churn-сигналов shared-доменов. Vercel и Netlify остаются лучшим хостингом в своём классе. Домен это адрес, который принадлежит проекту, а не арендуется у платформы. Без собственного домена любой SEO-трафик упирается в стеклянный потолок shared demotion.

Суверенитет проекта не ограничивается доменом. Без VPN, DNSSEC, hardened DNS и изолированных окружений для админ-доступа сайт уязвим для санкционных рисков, DDoS-атак и deanonymization администратора. Подробнее в этой статье: [[vpn-crypto-2026]]. См. также: [[sovereign-tech]].

### Q: Google плохо индексирует *.vercel.app?

Индексирует, но с задержкой и пониженным приоритетом. Shared-домены попадают под sandbox-классификатор: медленная индексация, нулевой шанс попасть в Discover и News. Переезд на собственный домен ускоряет индексацию в 3-10 раз по данным 2024-2026 годов.

### Q: Сколько стоит свой домен + SSL + DNS в 2026?

Домен .ru/.com: 600-1500 ₽/год. Cloudflare Registrar продаёт без наценки по цене регистратора. SSL бесплатно через Let's Encrypt или Cloudflare. DNS бесплатно через Cloudflare. Итого: 600-1500 ₽/год без скрытых платежей.

### Q: Можно ли держать Vercel-хостинг со своим доменом?

Да, это правильная архитектура. Vercel Hobby Plan поддерживает custom domain бесплатно. Домен регистрируется отдельно, проксируется через Cloudflare, указывает на Vercel через DNS. Vercel отвечает за билд и edge-функции, домен за бренд и SEO.

### Q: Какой TTL ставить на DNS при переезде?

За 24 часа до переезда снизить TTL до 300 секунд (5 минут) на старых записях для ускорения распространения изменений. После стабилизации через 1-2 недели поднять TTL обратно до 3600-14400 (1-4 часа) для снижения нагрузки на DNS.

### Q: Бренд занят в .com — что делать?

Купить у текущего владельца (часто $1000-$100K). Использовать альтернативный TLD (.co, .io, .net, .design, .studio). Добавить префикс: getbrand.com, trybrand.com, brandhq.com. Переименовать бренд, если в SEO ещё не вложены годы. Для нового проекта сразу проверять доступность всех вариантов.
